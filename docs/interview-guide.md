# 项目面试讲解

## 三分钟讲解框架

PaperResearch Agent 是一个面向科研场景的论文 RAG 与深度研究系统。它与普通 PDF
聊天工具的区别有三点：第一，解析阶段保留章节、页码、边界框、表格和引用结构；第二，
检索链路同时使用 Dense 与 BM25，通过 RRF 融合并保存完整 Trace；第三，LangGraph
工作流会判断证据缺口，在预算允许时继续外部搜索和导入，并对最终报告做引用校验。

项目用 10 篇真实论文完成了解析、切分和检索消融。结构化 Dense 的 Hit@1 为 0.80，
固定切分只有 0.30；Hybrid 的 Recall@10 为 0.90。当前词法 Rerank 会降低指标，因此
没有把它包装成提升，而是保留为可替换基线，后续接 Cross-Encoder。

## 高频问题

### 为什么不用固定长度切分？

固定切分会破坏章节和实验语义边界。结构化策略先按论文结构和内容类型组织，相邻同类块
合并，只有超长块才按 Token 切分。消融显示结构化 Dense 明显优于固定 Dense。

### 如何控制幻觉？

答案和分析字段必须绑定证据块；低于阈值时拒答；最终报告以论文、章节和页码标注引用，
并检查引用标记是否能映射回检索证据。

### 为什么使用 RRF？

Dense 与 BM25 的分数空间不可直接比较，RRF 只依赖名次，能稳定融合语义召回与专有名词
召回，同时保留双路排名供审计。

### Agent 为什么不会无限循环？

ResearchState 中显式记录迭代、外部搜索、论文、证据、估算 Token 和无新增证据轮数，
条件边在每轮缺口判断后检查预算，而不是依赖模型自行决定停止。

### 当前最大不足是什么？

Embedding、Rerank 和综合报告仍使用本地确定性基线；50 条评测集尚未由领域标注者人工
复核；Tesseract、GROBID 和真实 Docker 服务需在完整运行环境做端到端验收。
