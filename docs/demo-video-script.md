# 三分钟演示视频脚本

## 0:00–0:25 项目定位

画面：控制台和系统架构图。

旁白：PaperResearch Agent 不是普通 PDF 聊天工具。它保留论文结构和页码，使用混合检索
与引用校验，并通过 LangGraph 自动判断证据缺口、搜索论文和生成研究报告。

## 0:25–1:05 PDF 解析与单篇分析

画面：上传论文、状态变化、论文详情页。

旁白：上传后系统计算 SHA-256 去重，通过 ParserRouter 选择解析器，输出结构块、页面图片
和审计报告。自动分析的研究问题、方法和实验字段全部绑定原文 block 和页码。

## 1:05–1:40 混合检索与评测

画面：检索 Trace 和评测中心。

旁白：Dense 和 BM25 并行召回，通过 RRF 融合并保存每个阶段的排名。消融显示结构化
Dense Hit@1 为 0.80，固定切分为 0.30；Hybrid Recall@10 为 0.90。

## 1:40–2:35 深度研究

画面：提交主题、节点轨迹、最终报告。

旁白：LangGraph 将问题拆成四类子问题，先查本地证据；存在缺口时才调用外部搜索和可选
自动导入。多维预算限制循环，最终报告对引用逐条校验。本次审计引用有效率为 20/20。

## 2:35–3:00 工程化与边界

画面：Docker Compose、CI、发布清单。

旁白：项目支持容器化部署、统一错误请求 ID、自动测试和可重复评测。当前 50 条数据仍是
银标，必须人工复核后才能作为正式指标；生产环境还应替换 BGE-M3 和 Cross-Encoder。
